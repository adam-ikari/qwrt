qwrt

An embeddable QuickJS-ng runtime wrapper in C99. WinterCG-compatible JS polyfill, a Platform Abstraction Layer, and native extensions — all from source, no system libraries.

View on GitHub API Reference

Why qwrt

Strict C99

Embeddable in any C99 codebase. No C11 requirement on the host; vendored C11 deps (QuickJS-ng, libuv) are isolated.

No system libs

QuickJS-ng, mbedTLS, miniz, libuv all built from source via CMake add_subdirectory. Fully self-contained.

WinterCG polyfill

fetch, console, crypto.subtle, streams, timers, fs, URL, TextEncoder — 21 modules, precompiled to bytecode.

Portable PAL

libuv (Linux/macOS), FreeRTOS (ESP32-S3), mock (tests). Same code, three platforms.

Multi-context

Spawn/suspend/resume isolated JS contexts in one runtime, each with its own PAL and permissions.

No globals

Zero mutable file-scope state. Per-runtime isolation; safe to run multiple qwrt_t instances.

Quick start

git clone --recursive https://github.com/adam-ikari/qwrt.git
cd qwrt
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(nproc)

Minimal example

#include <qwrt/qwrt.h>
#include <pal_uv.h>

int main(void) {
    qwrt_pal_t *pal = pal_uv_create(NULL);
    qwrt_t *rt = qwrt_create(&(qwrt_config_t){ .pal = pal });

    qwrt_eval(rt, "console.log('Hello from QuickJS!');", NULL);

    char *result = NULL;
    qwrt_eval(rt, "1 + 1", &result);
    printf("1 + 1 = %s\n", result);   /* "2" */
    qwrt_free(result);

    while (pal->run_cycle(pal, 100) > 0) qwrt_tick(rt);

    qwrt_destroy(rt);
    return 0;
}

Build options

Feature toggles are prefixed QWRT_WITH_* (TLS, COMPRESS, CRYPTO_EXT, TEXTCODEC, WASM3/WAMR); PAL backends use QWRT_PAL_* (UV, MOCK, FREERTOS). See the README for the full list.
